Описание занятия:

 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout [файл]  - для конкретного файла;
git checkout .  - для всех фалов;

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged [файл] - для конкретного файла;
git restore --staged . - для всех фалов;

3.Как отменить ошибочный commit ?
git commit --amend -m "комментарий к коммиту" - для изменения последнего коммита;
git reset HEAD~n - откатиться назад на n коммитов;

4.Как создать новую ветку и переключится в нее?
git checkout -b [название ветки] 

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
 В первом случае изменения происходят только в одной ветке, а во втором  - изменения происходят в двух ветках  и в результате создается новый коммит, содержащий в себе изменения двух веток. При этом ветка "main" перемещается на этот новый коммит, а другая ветка остается лишь со своими последними изменениями.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
Когда в каждой ветке сделан коммит одного и того же файла.  Для разрешения конфликта, в редакторе нужно выбрать предпочтительный вариант изменений в файле и сделать коммит уже измененного файла.

7.Можно ли удалить ветку? Как?
можно командой   git branch -d [имя ветки]
